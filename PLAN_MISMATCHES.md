# PLAN_MISMATCHES — 设计偏差记录

本文件记录固件实现相对 `esp32-plan.md` 的所有偏差：规范未定而自行设计的部分，
以及与现状冲突需修正的部分。每条含：对应章节 / 原描述 / 缺失或冲突 / 采用设计 /
原因 / 未来正式规范需改处。本文件随工程演进保持同步，过时条目删除。

---

## 1. 目标芯片：esp32 → esp32s3

- 章节：一.1 主控「ESP32-S3」
- 原描述：正文多处声明 ESP32-S3。
- 冲突：仓库原 `sdkconfig`/`build` 配的是普通 esp32（`CONFIG_IDF_TARGET="esp32"`）。
- 采用设计：删除旧 `sdkconfig`/`build`，新增 `sdkconfig.defaults` 固化
  `CONFIG_IDF_TARGET="esp32s3"`，执行 `idf.py set-target esp32s3`。
- 原因：规范明确要求 S3，旧配置为残留。
- 未来正式规范需改处：无（规范本身正确，修正的是工程配置）。

## 2. 电机接口：未定义 → RS485 Modbus RTU

- 章节：一.3「机构自由度、传动与电机参数由固件本地维护，本计划不约定」。
- 缺失：未定电机控制接口（脉冲/方向 还是 总线）。
- 采用设计：RS485 半双工 + Modbus RTU（19200 8N1，功能码 0x03/0x06/0x10），
  YZ-AIM 手册协议；两台共用一条总线，从站地址区分。
- 原因：YT42B2 同时支持脉冲+方向与 485；485 单总线接两台电机布线最简、可读回位置/报警。
- 未来正式规范需改处：一.3 补「电机接口：RS485 Modbus RTU」。

## 3. 逆运动学几何：按机构定义 → 参数化占位

- 章节：二「坐标系」、一.3「传动参数由固件本地维护」。
- 缺失：未给臂长、关节零位、减速比、连杆几何。
- 采用设计：`kinematics` 用占位模型——A = `atan2(y, x)` 对准目标水平方位角，
  B = 目标深度 z；几何常量集中在 `app_config.h` 并标 TODO，待现场标定替换。
- 原因：无机构尺寸无法建真实逆解；占位模型保证接口完整、可编译、可联调框架。
- 未来正式规范需改处：二 补真实机构几何参数与逆解模型，删除占位说明。

## 4. 动作点叠加方式：按机构定义 → 自定义叠加规则

- 章节：七「approach / insert / retract 三个点由目标点叠加参数算出，具体叠加方式按机构定义」。
- 缺失：未给叠加方向与符号。
- 采用设计：A 轴恒为方位角；B 轴（深度，向下为正）——
  approach = `z − approach_height`，insert = `z + insert_depth`，retract = `z − retract_height`。
- 原因：z 向下为正（二），接近在目标上方、插入到目标下方、拔出回上方，符合除草杆插拔语义。
- 未来正式规范需改处：七 补明确的叠加公式与符号约定。

## 5. SUM 算法：示例占位 1C → 二进制累加（plan 已声明以累加为准）

- 章节：五「校验算法」。
- 冲突：Jetson plan 第六节示例把 SUM 写成 `1C`，本规范累加算法得 `43`。
- 采用设计：实现严格用累加算法（`protocol_checksum`），可复现示例
  `AA5501002A00EBFFEE00400043` 自测通过。
- 原因：规范明文「固件以本累加和算法为准」，且与 Jetson `checksum()` 实现完全一致。
- 未来正式规范需改处：上位机 plan 第六节示例帧 SUM 占位值 `1C` 应更正为 `43`。

## 6. STOP 行为：由固件定义 → 锁定当前位置 + 回 ERROR(0x10)

- 章节：八「STOP 后若动作未完成，按 ERROR 或停在安全位姿，由固件定义并据实回报」。
- 缺失：未定 STOP 的具体停车方式与回报。
- 采用设计：STOP 在段边界检查；触发时把两台电机目标设为各自当前位置（锁定急停），
  状态置 STOPPED，回 `ERROR(ERR=0x10)`。
- 原因：规范允许固件定义；段边界停车避免段内中断造成位置不确定，回 ERROR 让上位机记失败。
- 未来正式规范需改处：八 补 STOP 具体语义（锁定位置 + ERROR 0x10）。

## 7. 错误码取值：未给 → 自定义 0x01/0x02/0x03/0x04/0x10

- 章节：四「错误码用于现场排查」、六「ERR = 错误码」。
- 缺失：未给错误码取值表。
- 采用设计：0x01 超工作空间 / 0x02 电机通信失败 / 0x03 电机报警 / 0x04 运动超时 / 0x10 STOP 中止。
- 原因：覆盖逆解、通信、电机、超时、人工中止五类失败，便于现场区分。
- 未来正式规范需改处：四 补错误码定义表；需要时约定上位机解析。

## 8. Modbus 从站地址：未给 → A=1 / B=2

- 章节：项目情况「A 电机控水平旋转，B 电机控垂直平面运动」。
- 缺失：未给 485 从站地址。
- 采用设计：A=1（水平旋转）、B=2（垂直/插拔），共用一条 RS485 总线。
- 原因：规范只给职责未给地址，需指定才能寻址；按职责顺序编号。
- 未来正式规范需改处：补从站地址分配（或由 app_config 维护，规范不约定）。

## 9. 引脚分配：未给 → app_config 占位 + TODO

- 章节：一.2 串口、一.3 机构。
- 缺失：未给 GPIO 引脚。
- 采用设计：`app_config.h` 占位——Jetson UART1 TX17/RX18；RS485 UART2 TX4/RX5/DE6，均标 TODO。
- 原因：无硬件接线图，占位保证可编译；现场按实际接线改一处即可。
- 未来正式规范需改处：补引脚分配表（或留 app_config 维护）。

## 10. Modbus 波特率：手册默认 19200 → 沿用 19200

- 章节：一.2「UART 115200 8N1」（指 Jetson 链路）。
- 缺失：电机链路波特率未在本规范定义。
- 采用设计：电机 RS485 用手册默认 19200 8N1，独立于 Jetson 115200 链路（两条独立 UART）。
- 原因：避免改电机内部波特率参数；两链路物理独立互不影响。
- 未来正式规范需改处：可补「电机链路 19200，Jetson 链路 115200」以免混淆。

## 11. 电机通信模式配置时机：启动自动配置 → 先读 0x0A 判断，避免重复写 flash

- 章节：一.3「电机参数由固件本地维护」。
- 缺失：未定通信位置模式的配置时机与驱动器输出使能。
- 采用设计：启动时先写「驱动器输出使能 0x01=1」（非保存类，每次上电须置 1 才出力）；
  再读电子齿轮分子 0x0A——若已为 0 视为已配置直接跳过；否则写「Modbus 使能 0x00=1 +
  分子 0x0A=0 + 保存 0x14=1」并标记 `need_reboot`（手册：保存类参数需重新上电生效）。
- 原因：手册寄存器表 0x01「驱动器输出使能」0=禁止输出，位置控制要出力必须置 1（已与用户确认）；
  每次上电都写 flash 会损耗寿命，读判断后只在未配置时写一次。
- 未来正式规范需改处：无（属固件内部实现细节）。

## 12. 关节零位：plan 未给 → 逆解脉冲换算减零位偏置

- 章节：二「坐标系」、一.3「传动参数由固件本地维护」。
- 缺失：未给关节零位定义。
- 采用设计：`kin_a_deg_to_pulses`/`kin_b_mm_to_pulses` 换算时先减 `CFG_ARM_A_ZERO_DEG`/
  `CFG_ARM_B_ZERO_MM`（占位 0.0），再乘减速比与每圈脉冲。
- 原因：电机绝对位置零点与机构关节零位通常不重合，换算须扣除偏置；现为占位，待现场标定。
- 未来正式规范需改处：二 补关节零位定义与标定方法。

## 13. 位置模式速度寄存器：沿用 0x02

- 章节：七「move_speed / insert_speed」。
- 缺失：未给速度写入的 Modbus 寄存器。
- 采用设计：`seg_move` 每段运动前写 0x02（目标/最大速度）设定该段运行速度。
- 原因：手册 0x02 标注「目标速度」，位置模式按此速度运行到目标位置（已与用户确认沿用 0x02）。
- 未来正式规范需改处：无（属固件内部实现细节）。

## 14. ESP32 不维护保活节拍参数

- 章节：九「上电与保活」。
- 冲突：plan 九给出 `ping_interval_ms`/`pong_timeout_ms`（默认 1000/3000），但这些是 Jetson 侧
  发 PING 与判离线的节拍，ESP32 只被动即时回 PONG。
- 采用设计：app_config 不保留 PING_INTERVAL/PONG_TIMEOUT/ACCEPTED_TIMEOUT，避免无效配置；
  ESP32 收到 PING 立即回 PONG，不计时。
- 原因：被动响应方持有对端节拍参数无作用，留之即死配置。
- 未来正式规范需改处：九 可注明这些节拍参数仅约束 Jetson 侧。
